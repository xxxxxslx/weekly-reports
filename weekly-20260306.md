### 学习周报（20260307）
#### 一、本周学习内容
| 学习主题 | 核心知识点 |
|----------|------------|
| 设置用户签名|`git config --global user.name/email` 配置全局用户签名|
| 初始化本地库 | `git init` 初始化本地仓库（生成 .git 隐藏目录） |
| 查看本地库状态 | `git status` 查看文件是否追踪/暂存/提交，区分工作区/暂存区状态 |
| 添加到暂存区 | `git add 文件名`/`git add .` 将工作区文件提交到暂存区 |
| 提交到本地库 | `git commit -m "提交备注" 文件名` 将暂存区文件提交到本地库，生成版本记录 |
| 修改文件 | 修改后 `git status` 会提示文件已修改，需重新 `add`+`commit` 提交新版本 |
| 版本穿梭 | 1. `git log`/`git reflog` 查看版本记录（reflog 更简洁）<br>2. `git reset --hard 版本号` 切换到指定版本 |
| 分支概述和优点 | 1. 分支本质是版本指针，默认主分支 `main/master`<br>2. 优点：并行开发、隔离功能、避免代码冲突 |
| 分支查看/创建/切换 | 1. `git branch` 查看分支（* 表示当前分支）<br>2. `git branch 分支名` 创建分支<br>3. `git checkout 分支名`/`git switch 分支名` 切换分支 |
| 分支合并（正常） | 1. 切回主分支 `git checkout main`<br>2. `git merge 分支名` 合并指定分支到当前分支（无冲突时直接合并） |
| 分支合并（冲突） | 1. 冲突标记：`<<<<<<< HEAD`（当前分支）/`=======`/`>>>>>>> 分支名`（待合并分支）<br>2. 手动修改冲突文件 → `add` → `commit`（无需文件名）完成合并 |
| 团队协作 | 1. 团队内：分支推送/拉取、合并<br>2. 跨团队：Fork 仓库 → Clone 到本地 → PR（Pull Request）提交流程 |
| GitHub 远程库&别名 | 1. GitHub 创建远程库（Public/Private）<br>2. `git remote add 别名 远程地址` 给远程库设置别名（默认 `origin`）<br>3. `git remote -v` 查看远程别名及地址 |
| 推送本地库到远程库 | `git push 远程别名 分支名`（如 `git push origin main`） |
 | 拉取远程库到本地库 | 1. `git pull 远程别名 分支名`（等价于 `git fetch + git merge`）<br>2. 解决拉取后的冲突，再提交 |
| 克隆远程库到本地 | `git clone 远程地址` 自动完成：下载代码 + 初始化本地库 + 设置远程别名 `origin` |

#### 二、学习总结
1. **重点掌握**：工作区→暂存区→本地库→远程库的核心流程，分支创建/合并/冲突解决，GitHub 远程协作基本命令。
