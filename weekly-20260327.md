## ORM

Python 与数据库的映射关系,如下：

|Python 类概念|对应数据库概念|示例说明|
|-|-|-|
|**类 (Class)**|表 (Table)|表示实体的结构定义|
|**属性 (Attribute)**|字段/列 (Column)|描述实体的具体特征|
|**实例对象 (Instance)**|记录/行 (Row)|具体的一条数据|
|**类方法**|存储过程/查询|数据的操作方法|

## 依赖注入

1. 依赖注入的作用
将公共逻辑（如数据库连接、登录验证、参数校验）抽离为独立函数，在路由接口中通过 Depends() 自动注入，实现：
* 代码复用
* 逻辑解耦
* 统一管理资源
2. 数据库会话依赖实践
通过 get\_db 异步依赖函数，自动创建并管理 AsyncSession，在所有接口中统一注入，避免重复开关连接、资源泄漏。

## 数据库交互流程

1. **数据库连接配置**
* 同步 SQLite：`sqlite:///test.db`
* 异步 SQLite：`sqlite+aiosqlite:///test.db`（必须安装 aiosqlite）
* 异步引擎：`create_async_engine`
2. **会话（Session）理解**
Session 是应用与数据库之间的一次对话，所有 ORM 操作必须在会话内完成，包括：
* 缓存待操作对象
* 执行事务提交/回滚
* 关闭数据库连接

`async with AsyncSessionLocal() as session: yield session`

## 增删改查

### 1\. 新增（Create）

* 创建模型对象  
`user = User(name="张三", age=20)`
* 添加到会话  
`db.add(user)`
* 提交事务  
`await db.commit()`
* 刷新获取最新数据  
`await db.refresh(user)`

### 2\. 查询（Read）

* 构建查询语句  
`stmt = select(User)`
* 条件过滤  
`stmt = select(User).where(User.id == 1)`
* 异步执行查询  
`result = await db.execute(stmt)`
* 获取多条数据  
`users = result.scalars().all()`

**扩展查询**

* 精确查询：`select(User).where(User.name == "张三")`
* 模糊查询：`select(User).where(User.name.like("%张%"))`
* 范围查询：`select(User).where(User.age >= 18)`
* AND 多条件：`select(User).where(and_(User.name=="张三", User.age==20))`
* OR 多条件：`select(User).where(or_(User.age<18, User.age>60))`
* 排序：`select(User).order_by(User.age.desc())`
* 分页：`select(User).offset(0).limit(10)`
* 统计总数：`select(func.count(User.id))`
* 非空查询：`select(User).where(User.age.isnot(None))`
* IN 查询：`select(User).where(User.id.in_([1,2,3]))`

### 3\. 修改（Update）

* 先查询对象是否存在  
`user = await db.get(User, user_id)`
* 直接修改对象属性  
`user.name = "新名字"`
* 提交事务  
`await db.commit()`

### 4\. 删除（Delete）

* 先查询对象  
`user = await db.get(User, user_id)`
* 删除对象  
`await db.delete(user)`
* 提交事务  
`await db.commit()`
