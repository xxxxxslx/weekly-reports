# Redis

### 一、Redis 定位

- **数据模型**：**Key-Value 键值对**。Key 是字符串，Value 支持丰富的**原生数据结构**：
  - 基础：String（字符串）、Hash（哈希）、List（列表）、Set（集合）、ZSet（有序集合）
  - 高级：Bitmap（位图）、HyperLogLog、GEO（地理坐标）、Stream（流）
- **存储介质**：**基于内存**（极快读写）
- **本质**：**内存型数据结构服务器**，而非文件型非结构化库

### 二、Redis归属
**半结构化/结构化键值**

- **有结构，但非表结构**：Value 是**强类型数据结构**（不是无格式 blob）
- 无表、无行、无列、无外键关联
- 归类：**键值型 NoSQL → 非关系型 → 半结构化/非结构化存储**

### 三、Redis 与典型非结构化数据库的区别

| 特性           | Redis（键值）            | MongoDB（文档/非结构化）       | 对象存储（S3/COS）     |
| :------------- | :----------------------- | :----------------------------- | :--------------------- |
| **核心模型**   | Key → 固定数据结构       | Key → BSON 文档（灵活 Schema） | Key → 文件/二进制      |
| **Value 结构** | 严格、强类型             | 松散、动态字段                 | 无结构（二进制流）     |
| **查询能力**   | 按 Key 精确查找          | 文档内复杂查询、索引           | 仅按 Key 检索          |
| **典型场景**   | 缓存、会话、计数器、队列 | 内容管理、用户数据、IoT        | 图片、视频、大文件存储 |


### 四、Redis数据结构对比
| 类型 | 特点 | 存储形式 | 典型应用场景 |
| :--- | :--- | :--- | :--- |
| **String**<br>字符串 | 最简单、最通用<br>可存数字、文本、二进制<br>支持加减计数 | key → value<br>单个键值对 | 缓存对象(JSON)<br>分布式锁<br>计数器、限流<br>简单KV缓存 |
| **Hash**<br>哈希 | 一个key对应多个field-value<br>适合结构化对象<br>可单独修改字段 | key → {field:value} | 用户信息<br>商品详情<br>配置项<br>多维属性存储 |
| **List**<br>列表 | 有序、可重复<br>两端进出快<br>链表结构 | key → [v1, v2, v3...] | 消息队列<br>栈/队列<br>时间线、动态流 |
| **Set**<br>集合 | 无序、自动去重<br>支持交并差集 | key → {v1, v2, v3} | 好友/关注列表<br>去重统计<br>共同好友、推荐 |
| **ZSet**<br>有序集合 | 按分数自动排序<br>不重复、有排名 | key → {member:score} | 排行榜<br>延时任务队列<br>热度排序 |

**Milvus** 是目前最主流的**开源向量数据库**（Vector Database），专为 AI 场景下的**海量高维向量**（Embedding）存储和**相似度检索**设计，和你之前问的 Redis 是完全不同定位的数据库。

# Milvus
### 一、定位
- **Redis**：内存 KV 数据库，擅长**精确查询、缓存、简单数据结构**（String/Hash/List/Set/ZSet）
- **Milvus**：专门存**AI 向量**（如大模型输出的 768/1024/1536 维向量），只做一件事：**找最相似的 Top-K**（语义检索、以图搜图）

### 二、适用场景（和 Redis 完全互补）
| 场景 | 说明 |
| :--- | :--- |
| **RAG 知识库检索** | 把文档转向量，用户提问先搜 Milvus 取相关片段给大模型 |
| **以图搜图 / 视频去重** | 图像特征向量相似度检索 |
| **语义搜索** | 超越关键词，理解意图（如“推荐轻松的旅行歌”） |
| **推荐系统** | 用户/物品向量，匹配兴趣相似 |
| **大模型长记忆 / 会话缓存** | 历史对话向量化，快速召回上下文 |
| **风控/异常检测** | 行为向量找异常样本 |

### 三、Milvus vs Redis
| 维度 | **Milvus** | **Redis** |
| :--- | :--- | :--- |
| **核心能力** | 向量相似度检索（ANN） | 精确 KV、缓存、简单数据结构 |
| **数据类型** | 高维向量（128~2048维） | 字符串、哈希、列表、集合等 |
| **查询方式** | Top-K 近似最近邻 | 精确匹配、范围、排序 |
| **规模** | 百亿~万亿向量 | 百万~千万级（内存受限） |
| **架构** | 分布式、存算分离、云原生 | 单机/主从/集群，内存优先 |
| **典型组合** | Milvus 做向量召回 → Redis 做热点缓存 & 会话 | 一起用：**Redis 热层 + Milvus 冷层** |
