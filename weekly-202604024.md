#### 函数作为参数 ★
```go
// 定义函数类型
type Operation func(int, int) int

// 函数作为参数
func calculate(op Operation, a, b int) int {
    return op(a, b)
}

// 使用
add := func(a, b int) int { return a + b }
result := calculate(add, 5, 3)  // 8
```

#### 函数作为返回值 ★
```go
func makeMultiplier(factor int) func(int) int {
    return func(x int) int {
        return x * factor
    }
}

double := makeMultiplier(2)
fmt.Println(double(5))  // 10
```

#### 闭包★★
```go
// 闭包 = 函数 + 其引用的外部变量
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

c1 := counter()
fmt.Println(c1()) // 1
fmt.Println(c1()) // 2
fmt.Println(c1()) // 3

c2 := counter()   // 新的闭包，独立的 count
fmt.Println(c2()) // 1
```


#### defer 延迟执行★
```go
// 1. 资源释放
func readFile(path string) error {
    file, err := os.Open(path)
    if err != nil {
        return err
    }
    defer file.Close()  // 函数结束时执行
    
    // 处理文件...
    return nil
}

// 2. defer 执行顺序（LIFO - 后进先出）
func deferOrder() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
    // 输出: 3 2 1
}

// 3. defer 与返回值
func deferReturn() (result int) {
    defer func() {
        result++  // 可以修改命名返回值
    }()
    return 0  // 实际返回 1
}
```

#### panic 和 recover
```go
// panic: 程序崩溃
func riskyOperation() {
    panic("出现严重错误")
}

// recover: 捕获 panic
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("捕获到 panic:", r)
            // 可以进行日志记录、清理等操作
        }
    }()
    
    riskyOperation()  // 这里会 panic，但会被捕获
    fmt.Println("这行不会执行")
}

// 实际应用
func divide(a, b int) (result int) {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("除法错误:", r)
            result = 0
        }
    }()
    
    if b == 0 {
        panic("除数不能为零")
    }
    return a / b
}
```


| 机制 | 使用场景 |
|------|---------|
| defer | 资源释放、锁解锁、文件关闭 |
| panic | 不可恢复的严重错误 |
| recover | 顶层错误处理、防止程序崩溃 |

## 重点难点总结



### 闭包
- **本质**：函数绑定了外部变量，形成独立环境，变量会从栈逃逸到堆
- **循环陷阱**：循环变量被所有闭包共享，解决方式是传参或创建局部副本 `i := i`
- **内存风险**：闭包持有大对象会导致无法垃圾回收，用完应设为 `nil` 释放
- **goroutine 注意**：协程闭包同样会捕获循环变量，必须通过参数传递值副本

---

### defer
- **执行时机**：函数 return 前执行，多个 defer 按**后进先出**顺序
- **参数求值**：普通参数**注册时就确定值**，闭包参数**执行时才取值**
- **返回值修改**：只能修改**命名返回值**，匿名返回值改不了
---

### panic/recover
- **使用原则**：panic 仅用于不可恢复的严重错误，日常优先用 error 处理
- **生效范围**：recover 只在 defer 函数内有效，且只能捕获**当前 goroutine** 的 panic
- **恢复限制**：recover 后仅恢复当前调用栈，程序状态可能已不一致，谨慎继续执行

---

### 函数参数
- **全值传递**：Go 没有引用传递，所有参数都是值拷贝
- **slice/map/chan**：传的是描述符拷贝，但底层数据共享，所以改元素会影响外部
- **append 特殊性**：slice 的 append 可能触发扩容换底层数组，外部看不到新元素
- **性能**：大结构体传指针避免拷贝，小类型直接传值更快
