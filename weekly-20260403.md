## 切片

### （一）基础
定义：切片是对数组的抽象，属于**引用类型**，本身不存储数据，底层依赖数组实现。  
其结构包含三个核心部分：指针（指向底层数组）、长度（len，当前元素个数）、容量（cap，底层数组可容纳的最大元素个数）。

3.  与数组的核心区别   

    | 特性 | 数组 | 切片 |
    |------|------|------|
    | 长度 | 固定，声明时必须指定 | 可变，可动态扩容 |
    | 类型 | 值类型，赋值/传参复制整个数组 | 引用类型，赋值/传参仅复制切片本身 |
    | 声明格式 | [n]int（n为固定长度） | []int（不指定长度） |

### （二）操作

#### 1. 增删改查
```
// 追加元素（append是核心，返回新切片）
s := []int{1,2,3}
s = append(s, 4) // 追加单个元素
s = append(s, 5, 6) // 追加多个元素
s = append(s, []int{7,8}...) // 追加另一个切片

// 删除元素（无内置函数，通过append拼接实现） 
index := 1 // 要删除的元素下标
s = append(s[:index], s[index+1:]...) // 删除下标为1的元素
// 简化操作：删除开头（s = s[1:]）、删除末尾（s = s[:len(s)-1]）

// 修改元素（直接通过索引赋值）
s[0] = 100 // 将第一个元素改为100

// 查询元素（通过索引访问，越界会报错）
fmt.Println(s[2]) // 访问下标为2的元素
```
#### 2. 遍历
```
// for range（推荐）
for i, v := range s {
    fmt.Printf("s[%d] = %d\n", i, v)
}
```

#### 3. 拷贝（copy，深拷贝，避免引用影响）重要，否则同一个
```go
src := []int{1,2,3}
dst := make([]int, len(src)) // 必须提前分配空间，否则无法拷贝
copy(dst, src) // 拷贝src到dst
dst[0] = 999 // 修改dst，不影响src
```

### （三）切片底层
1.  扩容机制：
    - 触发条件：当 `len(s) > cap(s)` 时，触发扩容
    - 扩容规则：len < 1024 时，**翻倍扩容**；len ≥ 1024 时，**扩容1/4**
    - 注意：扩容后会创建新的底层数组，原切片与新切片断开引用，修改新切片不会影响原切片

2.  引用特性：
    - 未扩容时：切片与底层数组共用内存，修改切片元素会**影响原数组**
    - 已扩容时：切片指向新的底层数组，修改切片元素**不影响原数组**
    - 切片作为函数参数：传递的是引用，函数内修改切片元素会影响原切片；但如果函数内append触发扩容，原切片不受影响

3.  常见易错点：
    - 空切片 vs nil切片：二者len和cap均为0，但nil切片（var s []int）底层指针为nil，空切片（s := []int{}）底层指针非nil，二者均可直接append
    - 切取切片时：左闭右开，如arr[1:4]，包含arr[1]、arr[2]、arr[3]，不包含arr[4]
    - copy函数：必须提前给目标切片分配空间（make），否则无法完成拷贝

### （四）其他应用场景
1.  切片存切片：`[][]int`（如二维切片，存储多行多列数据）
2.  切片存map：`map[string][]int`（如分组存储，key为分组名，value为对应切片）
3.  切片与函数结合：将切片作为函数参数/返回值，实现批量处理（如批量修改切片元素、筛选切片内容）
4.  切片排序：使用sort包或自定义排序算法（冒泡、快速排序等）
